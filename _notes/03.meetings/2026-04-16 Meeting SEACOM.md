# 2026-04-16 Meeting SEACOM

**Data:** 2026-04-16 | **Tipo:** SEACOM | **Partecipanti:** Stefano, Cadigia, Nicola, Alice, Lorenzo, Andrea

## Riepilogo

Meeting di allineamento con Seacom focalizzato sulla pianificazione dell'installazione del cluster di produzione (target start settimana del 4-6 maggio 2026, completamento entro fine giugno). Sono state prese decisioni architetturali su registry immagini (GitLab interno), database su Kubernetes (Longhorn strict locality come punto di partenza), backup (Velero + S3/Minio), Keycloak autonomo e certificati dominio dataliguria.it. Seacom ha effettuato una demo dell'API OpenMetadata per il recupero di metadati delle tabelle, con prossimo step uno script di integrazione dbt. Discussione su fusi orari nei log (soluzione via Fluent Bit) e su utilizzo di un issue tracker GitLab per le attività residue.

## Punti Chiave

- **Pianificazione installazione**: start settimana 4-6 maggio, obiettivo fine giugno (6-8 settimane); Seacom prepara piano dettagliato con date
- **GitLab VM**: macchina virtuale esterna al cluster per GitLab, dimensionata con disco aggiuntivo per registry (~100GB iniziali)
- **Registry immagini**: registry interno GitLab con fallback su cloud registry; evita problemi fetch da registry pubblici
- **Minio storage**: le 4 macchine dedicate rimangono per Minio; storage sufficiente per 5 anni, non si considera soluzione hardware/software esterna nell'immediato
- **Database su K8s**: Longhorn con strict locality come opzione preferita; SAN da verificare ma improbabile per tempi; local path come alternativa
- **Backup**: Velero con destinazione S3 su Minio per applicativi K8s; GitLab backup su S3 (non NFS)
- **OpenMetadata API**: funziona il recupero di metadati (colonne, constraints, FK) via API; prossimo step: script per integrazione con dbt sources YAML
- **3 ambienti**: Runtime (produzione), Sviluppo (data engineers), Staging/infrastruttura (aggiornamenti, non sempre acceso)
- **Server fisici**: configurazione ILO completata entro mercoledì, poi si procede con installazione

## Decisioni Prese

- **Registry immagini**: GitLab interno con fallback cloud registry
- **OpenSearch**: installazione su cluster K8s
- **Minio**: installato sulle 4 macchine dedicate storage
- **Database**: Longhorn strict locality su K8s; verifica SAN ma si parte senza
- **Certificati**: dominio dataliguria.it
- **Backup**: S3 Minio come destinazione (Velero per applicativi, backup nativo GitLab)
- **Keycloak**: istanza autonoma su K8s gestita da Seacom; federazione LDAP futura
- **OpenMetadata job TTL**: 24h è accettabile (default), non è un problema
- **Timestamp log**: conversione timezone via Fluent Bit (da implementare a lungo termine)
